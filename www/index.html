Description

vss2svn is a Perl script to migrate revision history from a Microsoft Visual SourceSafe (VSS) database to the Subversion version control system. It will migrate all revision history, including comments, author names, and optionally revision dates (migrating dates uses a not-officially-supported feature of Subversion and can only be done if you are migrating to a brand-new Subversion repository).

This project has nothing to do with the vss2svn.py Python script which is floating around. Its methodology is loosely based on the migrate.pl script I found after some googling; however, I was not able to get that script to work properly and decided to start from scratch rather than try to modify it.

Status

This project is still considered Alpha quality, meaning it is still very much a work in progress, and anything can and will change. However, the basic functionality of this program is already in place, and several large VSS databases have already been successfully converted.

This script works by querying the existing VSS database for all changes ever made, arranging those changes in chronological order, checking them out of VSS one at a time, and then checking them in to Subversion. Additionally, it attempts to create atomic commits where possible (even though VSS doesn't support this feature!) by treating VSS checkins with the same author, date, and comment as a single Subversion revision. As you can imagine, all of this is a rather slow process. However, given VSS's well-known penchant for corrupting and/or losing users' data, it's worth it to move to a real version control system!

Project Source Code

To check out project source code using svn, use the following command:

svn checkout http://vss2svn.tigris.org/svn/vss2svn/trunk vss2svn --username username

You can also browse the repository at http://vss2svn.tigris.org/source/browse/vss2svn/, which is always available from the "Subversion" link in the left-hand menu. WARNING: the latest code in /trunk may not be in a usable state at any given point in time. Unless you're interested in checking out the latest code for development reasons, please grab the latest zipfile from the release folder in the downloads section.

Downloading Instructions

Perl Script

You can download the Perl script from the Release folder of the Documents and Files section. See the INSTALL.txt file within the release for instructions on running the script.

Win32 Executable

Since this program is still considered Alpha quality, I highly recommend running the Perl script if at all possible. This will help you to help me to help you if there's a problem. :) However, if installing Perl isn't an option for you, or you want to get up and running as quickly as possible, I also provide the latest released version as a Win32 statically-built executable. This file is somewhat large (over a meg) because it includes the entire Perl interpreter and helper modules.

You can download the Win32 executable from the win32 folder of the Documents and Files section.

Where To Go For Help

If you need help running this script, or encounter errors, please go to the mailing list instead of emailing me directly. If you don't wish to subscribe to the list, you can still send an email to users@vss2svn.tigris.org even without subscribing, and ask that you be copied on any replies.

Another useful resource in deciphering the sometimes confusing VSS error messages you may encounter is available at http://msdn.microsoft.com/library/default.asp?url=/library/en-us/guides/html/vsorierrormessages.asp.
